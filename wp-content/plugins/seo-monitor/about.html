Platform Overview

This platform is an integrated SEO and content automation ecosystem built on two core WordPress plugins — SEO Monitor and Blog Queue — that work together as a closed-loop content intelligence and production system.

The platform connects to Google Search Console for real-time performance data, imports competitive intelligence from SEMrush, and uses AI to both identify content opportunities and produce publication-ready blog posts with full SEO optimization.

Site-Agnostic Design: All AI prompts, branding references, and configurations dynamically pull the site name via get_bloginfo('name'), making the entire platform portable across multiple properties without code changes.

Plugin: SEO Monitor

SEO Monitor is the intelligence and optimization engine. It continuously monitors every indexed page through Google Search Console, scores pages against a multi-factor algorithm, and automatically refreshes underperforming content using AI.

Data Collection & Intelligence

  • Google Search Console Integration — OAuth2 service account authentication. Daily snapshots of clicks, impressions, CTR, and position for every published page. Top 5 search queries stored per page for AI context injection.
  • Keyword Intelligence — Growing keyword database with 28-day trends (rising/stable/declining), fuzzy keyword-to-post mapping, cannibalization detection, and opportunity scoring (0-100).
  • Competitive Gap Analysis — SEMrush CSV imports with AI auto-tagging, programmatic deduplication, batch consolidation, and 90-day keyword cooldown tracking.
  • Google Autocomplete Expansion — Weekly enrichment of top keywords with long-tail variations via Autocomplete API.

Page Scoring Algorithm

Every monitored page is scored across four weighted dimensions:

Content Refresh Engine

Three refresh tiers with smart escalation:

Refresh Types

  • Full Refresh — Complete content rewrite with new outline, FAQ, schema, and meta
  • SEO Refresh — FAQ, schema, meta, keyword, and title only
  • Meta Refresh — Title and meta description only (CTR fixes)

AI Pipeline Steps

  1. Generate structured outline (6-8 sections)
  2. Write 2,000-2,500 word HTML content
  3. Meta description (140-155 chars)
  4. FAQ generation (5 entries, accordion format)
  5. JSON-LD FAQPage schema
  6. Focus keyword & SEO title (Rank Math)
Smart Escalation: If a meta-only refresh doesn't improve performance on the next analysis cycle, the system automatically escalates to a full content refresh.

Plugin: Blog Queue

Blog Queue is the content production engine. It manages a pipeline of blog topics from ideation through AI-powered creation to publication, with full keyword targeting and SEO optimization at every step.

AI Topic Generation

The topic generator analyzes multiple data sources to suggest unique, non-overlapping blog ideas:

  • GSC Content Gaps — Keywords with impressions but no dedicated post
  • Trending Keywords — Rising search queries from Google Search Console
  • Competitive Gap Keywords — Imported from SEMrush, filtered by category and volume
  • LSI Keywords — Semantically related variants for topic differentiation
  • Existing Content Audit — Cross-references all post titles and queued topics to prevent duplication

Filtering options: Gap category, volume tier (High Volume 1000+, Low Difficulty KD≤30, Quick Wins), focus area, and quantity (5-50). The category dropdown dynamically updates available counts based on the selected volume filter.

Content Creation Pipeline

Blog creation is a 4-step AJAX process, each step executing independently to avoid server timeouts:

StepActionOutput
1Generate Outline6-8 section structure with detailed bullet points
2Write Content2,000-2,500 word HTML article with citations, formatting, keyword injection
3SEO MetadataMeta description, focus keyword, SEO title (Rank Math)
4FAQ & Schema5 FAQs in accordion format + JSON-LD FAQPage schema (ACF fields)

Queue Processing & Distribution

  • Round-Robin Distribution — Items are distributed across gap categories in rotation. With 10 categories, the first 10 blogs created will each be from a different category — preventing clusters of same-topic posts from publishing simultaneously.
  • Daily Limits — Configurable cap (default 5, max 50) prevents runaway processing.
  • Manual Processing — "Process Queue Now" selects up to 25 items from the full queue using round-robin distribution across all categories.

Content Quality Standards

All AI-generated content across both plugins (new blogs, refreshes, product descriptions) follows strict quality requirements enforced through detailed prompt engineering.

Writing Requirements

  • Length: 2,000-2,500 words minimum; each H2 section 200-350 words
  • Formatting: Paragraphs, bullet lists, numbered lists, callout boxes (tip/info/warning/key takeaway), comparison tables, blockquotes — minimum 2 format types per section
  • Depth: No surface-level summaries. Specific examples, tool names, commands, step-by-step details, real-world scenarios, and comparisons
  • Tone: Human-sounding. Varied sentence length. No generic openers ("In today's rapidly evolving...") or marketing clichés

Authoritative Citations (3-5 per article, minimum)

Every article must reference multiple source categories. Sources must be varied — no organization cited more than twice per article.

Keyword Intelligence & Lifecycle

Keyword Sources

Discovery Sources

  • SEMrush Import — Competitive gap keywords with volume and difficulty
  • Google Autocomplete — Long-tail expansion of top performers

Intelligence Features

  • Opportunity Scoring — 0-100 composite score (volume, position, CTR gap, trend)
  • Cannibalization Detection — Multiple pages ranking for same query
  • Lost Keyword Tracking — Keywords that disappear from GSC flagged, not deleted

Technical Resilience

Timeout Prevention

All long-running operations are broken into batched AJAX chains to prevent nginx 504 gateway timeouts:

OperationStrategy
Blog creation4-step pipeline (outline, content, meta, FAQ/schema)
SEO Monitor queueStep-based processing with 10-min cron intervals
GSC data collection2-phase batch processing (50 posts per batch)
SEMrush importMulti-phase (parse → upsert batches)
Tag consolidation3-phase (dedup → AI batch → apply batch)
Keyword collectionBatched at 200 keywords per AJAX call
AI auto-tagging40 keywords per AI call with running progress counter

Data Integrity

  • Growing keyword intelligence (upsert pattern — never delete-rebuild)
  • Lost keyword detection (flagged as "lost," not deleted from database)
  • Keyword cooldown prevents over-optimization and cannibalization
  • Round-robin queue distribution prevents topic clustering
  • Top performer protection (pages with >5 clicks are never auto-refreshed)

Security

  • Nonce verification on all AJAX endpoints
  • Capability checks (manage_options / manage_woocommerce)
  • Prepared SQL statements throughout (no injection risk)
  • Input sanitization on all user-provided data
